# Project Context Index

`.context/` stores the "what" of UniHub Workshop: requirements, system state, diagrams, data assumptions, and project decisions.

Use these files as follows:

- `../REQUIREMENTS.md`: canonical assignment requirements and project scope.
- `unihub-logic.md`: concise UniHub business logic that must not be hallucinated or changed casually.
- `full-requirements.md`: original expanded requirement summary already kept in the project context.
- `system-requirements.md`: short feature and technical-problem checklist.
- `architecture.md` and `architecture-skeleton.md`: architecture notes.
- `data_models.md`: data model notes.
- `decisions.md`: architectural decisions.
- `progress.md`: implementation progress.
- `roadmap.md`: next work items.
- `diagrams/workshop-flow.mmd`: Mermaid workshop registration and check-in flow.

Implementation-specific guidance belongs in `.codex/skills/`, not here.
